Javascript 101

Posted 20 February 2022


So what exactly is Javascript...?

I talked a bit about HTML and CSS last week, but two languages are not enough for the internet! This week we've been learning all about Javascript.

Imagine your dream car. It probably comes in lots of different colours, and you can probably pay extra for leather seats. But the underlying mechanics are the same no matter whether you go for Firebrick Red or Midnight Blue. That's basically what HTML is - the underlying structure of a website. It doesn't look pretty, but without it the website would just be a blank page. CSS is like the shiny paint job and those heated leather seats you always wanted. It sits on top of HTML and turns your website from boring to brilliant.

Now we know that HTML and CSS give us the tools to build a website that's full of content and looks pretty, you may be wondering what the point of Javascript is.

Let's go back to our car analogy. What modern car isn't full of electronics that control pretty much everything? That's what Javascript is. It's the website equivalent of the behind the scenes systems that let you change the temperature of those heated seats. Or maybe stop your car from skidding if you brake too hard. Depending on your priorities.

🚗

OK, you've convinced me that Javascript might be useful, but what can it do that those other languages can't?

What sets Javascript apart from HTML and CSS is that it lets you add dynamic content to your website. By that I mean things that change. Like forms and buttons, or content that updates automatically based on something external.

One of the tools that lets us do this is called 'control flow'. Which is a fancy way of saying, if this statement is true, do this thing. Otherwise do this other thing. A real word example would be figuring out what to do for dinner. My thought process would go something like this:

Can I be bothered cooking?

  • If the answer is no, order pizza.
  • Otherwise, decide on a recipe.

Do I have all the ingredients?

  • If not, go to the supermarket.
  • If yes, start cooking.

Does the recipe work?

  • If yes, enjoy a delicious home-cooked meal.
  • Otherwise, order pizza.

And so on...

As well as the if/then kind of logic, Javascript also lets you loop through things. You can tell it to do something 10 times, or 100, or 10,000! Or until some condition is met. A real world analogy (keeping with the dinner theme) would be: put dirty dishes into the dishwasher until there are no dirty dishes left on the bench.

So who is this Dom guy you keep talking about?

The DOM is literally the most mind-blowing thing I've come across all week. It's a really powerful tool for looking under the hood of what makes websites tick.

The DOM even lets you play around with the website code (just on your own machine of course - it doesn't let you break the internet!) You can also view what a website would look like on other devices (like mobile phones and tablets), and access loads of other information about the website, which makes the DOM an extremely powerful tool for web developers.

Getting a bit technical here, but what's the difference between accessing data from arrays and objects?

An array is essentially a list. You can access an item in the array by referring to its location, which is called its index. The slightly tricky thing about indexes is that they start at zero, not one. For example:

let dinner = ["pizza", "pasta", "tacos", "curry", "fish & chips"]

The code dinner[1] would return "pasta".

Objects are a bit more complicated. An object is made up of key-value pairs. For example:

let mealPlan = {

monday: "pizza",

tuesday: "pasta",

wednesday: "tacos",

thursday: "curry",

friday: "fish & chips"

}

You can retrieve a value by using its key. For example, mealPlan.wednesday would return "tacos".

And finally, what are functions and why are they useful?

Functions are bits of code that can be reused over and over. They can be set up to take different inputs, which makes them very versatile. Functions save you from having to write the same thing out multiple times and make your code easier to follow. So basically functions are super useful and make your life a lot easier (once you get the hang of how to write them!)
