Best Practices for using Classes and IDs

Posted 15 February 2022


If, like me, you're new to HTML and CSS, you may be wondering what the heck is the difference between an ID and a class...?

Let's start at the very beginning. Most websites use two different types of code: HTML and CSS. The HTML code contains the content of the website (such as text, images and videos), and the CSS code controls the appearance. While it is technically possible to include the styling in the HTML code, this is not considered good practice for various reasons which I'm not going to get into here!

HTML code is made from a series of features called "elements". Each heading, paragraph or image is a separate element. When you are designing a website, you need to get your HTML code and your CSS code to talk to each other. One way to do this is to assign elements in your HTML code a class or an ID, and then reference that class or ID in your CSS code to specify the format you would like.

Difference #1 - Uniqueness

Here's an example of some classes and IDs that I used when creating this website:

Notice how the IDs (id="blog-homepage" and id="blog-container") are only used once. However, the classes (class="blog-tile", class="blog-title" and class="blog-img" are used multiple times. This is a key difference between IDs and classes.

IDs are unique and can only be used once. Also, each element can only have one ID. Therefore, IDs should be used for single elements that have a unique style. If you go to my Blog homepage, you will notice that it has different styling to the rest of the website, so it was appropriate for me to use IDs here.

Classes on the other hand can be used on as many elements as you like, and each element can have multiple classes.The classes in the screenshot above are used to set the style for each tile on the blog homepage. Each tile refers to the same classes, so that when the style of those classes is changed in the CSS code it will change the appearance of each tile.

Difference #3 - Specificity

What is specificity, you ask? Basically it's a fancy word for priority. One of the key features of CSS is that it allows you to set multiple conflicting styles for a given element, and it applies a series of rules to determine which style takes priority. Specificity is a key factor in determining which style will be used. The other factors won't be discussed here, but you can read more about them here.

IDs have higher specificity than classes. Therefore, if an element has both a class and an ID, any conflicts will be resolved by applying the style specified for the ID.

Both IDs and classes also have higher specificity than a type selector (such as <h1> [heading 1] or <p> [paragraph]). In the example CSS code shown above, the style settings for the blog-homepage ID override the background colour, and the style settings for the blog-title class override the text colour.

Difference #4 - IDs can be used to create links

Have you ever noticed that when you click a link that scrolls to a particular heading on a webpage, a # appears in the URL? Does that seem familiar? (Hint: check out the images under Difference #2.)

These links within a webpage are created using IDs! Using the code <a href="#id-name">...<a> in your HTML will create a hyperlink. When you click the hyperlink, the page will scroll to the element with the ID you specified. You can click here to see this in action!
